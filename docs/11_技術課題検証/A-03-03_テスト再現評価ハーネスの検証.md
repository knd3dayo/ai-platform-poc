## 検証で確認したいこと

### 1. 正常系

- 自律実行の結果を自動テストや静的検査で点検できること。
- trace_id、設定、入力条件から再現実行の足掛かりを再構成できること。
- 書き込み系や高自律タスクでは、成果物レビューまたは PR ベースの承認ゲートへ接続できること。

### 2. 異常系

- 自動テストや成果物レビューを経ずに高リスク成果物を昇格しないこと。
- 再現条件が不足したまま品質判定や承認を行わないこと。
- テストが通らない、またはレビュー根拠が不足する成果物を黙って accept しないこと。

### 3. 運用系

- どのテスト、どのレビュー観点、どの承認者が結果判定に関与したかを追跡できること。
- 自動テスト、再現実行、成果物レビューの責務分担を文書化できること。
- A-03-02 の停止条件と矛盾しない形で評価ハーネスを配置できること。

## 対象構成

| 観点 | 主な既存実装 / 入口 | 備考 |
| --- | --- | --- |
| 自動テスト | pytest、回帰 test、smoke test | 実装ごとの最小品質保証 |
| 再現実行 | trace_id、config、audit log、resume 契約 | 実行経路の再現に利用 |
| 成果物レビュー | PR、差分確認、承認記録 | 高リスク成果物の昇格ゲート |
| 対象実装 | coding-agent、DeepAgents、SV/自律型入口 | A-04-05 / A-04-06 と接続 |

## 前提条件

- 自律実行結果に trace_id、設定ファイル、audit log などの再現材料が残ること。
- 自動テストまたは成果物レビューへ接続できる実行形態を利用すること。
- 書き込みや副作用を伴う成果物について、承認主体と承認単位を定義できること。

## 判定基準

| 観点 | 判定基準 |
| --- | --- |
| 構造成立性 | 自動テスト、再現実行、成果物レビューの接続点が文書化されている。 |
| 制御成立性 | テスト未通過、レビュー未実施、再現材料不足の成果物を accept しない条件を説明できる。 |
| 運用成立性 | trace_id、audit log、設定、差分レビューを用いて結果判定の根拠を再構成できる。 |
