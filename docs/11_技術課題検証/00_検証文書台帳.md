# 検証文書台帳

## 目的

本台帳は、[技術課題と対応方針](../03_検証準備/技術課題と対応方針.md) で定義したサブ課題に対応する検証文書を対象に、

* どの検証文書が存在するか
* 既存の検証文書がどの程度まで検証結果を整理できているか
* まだ起票されていない検証対象のサブ課題は何か
* 新規文書をどの命名規則で作るか

を一元的に確認するための文書である。

本台帳では、実装完了ではなく次の 2 つを分けて扱う。

* 起票状況: 検証文書が存在するか
* 検証状況: 検証文書に基づいて、結果がどこまで整理・判定されているか

## ステータス定義

| ステータス | 意味 |
| --- | --- |
| 文書あり | サブ課題に対応する検証文書が存在し、当該サブ課題との対応関係が明記されている。 |
| 未作成 | サブ課題に対応する検証文書が未作成である。 |

### 検証状況ステータス定義

| ステータス | 意味 |
| --- | --- |
| 確認済み | 検証文書に結果記録欄があり、主要観点の判定が概ね完了している。 |
| 一部確認済み | 検証は実施済みだが、未実装項目や追加確認項目が残っている。 |
| 結果未整理 | 文書はあるが、台帳に転記できる粒度で最終判定がまだ揃っていない。 |

## 命名規則

新規のサブ課題検証文書は、原則として次の命名規則とする。

* `サブ課題ID_短い論点名の検証.md`
* 例: `A-03-02_停止条件と予算上限の検証.md`
* 1 つの文書で複数サブ課題を扱う場合は、最も主となるサブ課題 ID をファイル名に使い、本文冒頭に副次対象を明記する。

共通の記載骨子は [99_サブ課題検証テンプレート.md](./99_サブ課題検証テンプレート.md) を利用する。
