# データカタログ主導による、高精度な自動開発・ドキュメント生成基盤の検証

## 1. 開発テーマ
**データカタログ主導による、高精度な自動開発・ドキュメント生成基盤の検証**
～ローカル検索・雑多なツール依存からの脱却と、セマンティックレイヤーによるコンテキスト提供の優位性証明～

## 2. 背景・目的
現在、多くのAIコーディングエージェントは「ローカル環境へのGitリポジトリのクローン」や「雑多なMCPツールを用いたファイル検索（grep等）」によってコンテキストを収集し、コード生成を行っている。しかし、エンタープライズのデータ領域において、真に価値のある情報（データの意味、鮮度、他部署での利用ルールなど）がローカルのソースコード内に完結していることは稀である。
従来のアプローチでは、エージェントが断片的なローカル情報から「推測」でコードを書くため、業務要件との乖離や手戻りが多発する。

本研究開発では、データ統合製品（データカタログ / セマンティックレイヤー）をAIの主たる情報源として連携させるアーキテクチャを構築する。そして、「従来型のローカルファイル依存エージェント」と「データカタログ主導エージェント」の出力品質を比較検証し、後者の優位性（正確性・網羅性）を証明することを目的とする。

## 3. 課題と解決アプローチ
既存のコーディングエージェント適用には、アーキテクチャの構造上、以下の限界が存在する。
* **コンテキストのサイロ化と推測への依存:** 物理テーブルのスキーマ（DDL等）がローカルにあっても、それが「ビジネス上どういう意味を持つ指標か」はコード化されていないことが多い。結果として、AIは一般的な知識で推測（ハルシネーション）してしまい、誤ったSQLやプログラムを生成する。

* **情報収集プロセスの非効率性:** 「データがどこにあるか」を探すために多数の汎用検索ツール（MCP）を無秩序に呼び出すため、コンテキストウィンドウを無駄に消費し、肝心の推論能力が低下する。

本検証では、以下のデータ・アーキテクチャ的アプローチで解決を図る。

* **データカタログ / セマンティックレイヤーからの直接コンテキスト注入:**
    Snowflake、Databricks、Denodo等のデータ統合製品を介して、組織で合意された「業務的な指標定義」「テーブル間のリレーション」などの構造化メタデータをAIへ一括提供する。

* **「探す」から「理解する」へのパラダイムシフト:**
    AIにファイルシステムを探索させるのではなく、データカタログという「意味の辞書」を直接参照させることで、推測を排除し、事実（ファクト）に基づくコード開発とドキュメント作成を実現する。

## 4. 検証スコープ：比較検証による優位性の証明
本検証では、以下のテーマに沿って「従来型アプローチ」と「提案アプローチ」の比較（A/Bテスト）を行い、優位性を定量・定性的に評価する。
※本検証は開発手法の優位性証明を主眼とするため、実行環境のサンドボックス化や監査ログ等の「生成AIガバナンス」領域の検証はスコープ外とする。

* **検証1: コンテキスト収集手法の比較（SQLおよびドキュメント生成）**
    * **[従来型]:** ローカルファイル（DDLや既存のクエリ集）とファイル検索ツールのみを与えたエージェントによる生成。
    * **[提案型]:** データカタログ（セマンティックレイヤー）のメタデータAPIに接続したエージェントによる生成。
    * *評価点:* 両者に対し同じ業務要件を提示し、「指標定義の正確な解釈」「不正なテーブル参照の有無」「生成されるデータ仕様ドキュメントの網羅性」を比較する。

* **検証2: 複雑なプログラム生成における手戻り回数の測定**
    * データの抽出・加工・可視化準備を含むPythonプログラム等の生成において、一発で期待する要件を満たすコードが生成されるか、またはプロンプトによる修正指示（手戻り）が何回発生するかを計測し、開発効率の向上度合いを検証する。

## 5. システム構成案（アーキテクチャ）
本検証では、比較対象として2つの環境を用意し、出力結果を評価する。

1. **提案環境：データカタログ主導型エージェント**
    * **データカタログ / セマンティックレイヤー:** 業務データの意味、メタデータを管理し、API/MCP経由で構造化されたコンテキストとしてAIに提供する。

    * **メタデータ連携エージェント:** データカタログから得た「意味」を元に、SQL、Pythonスクリプト、およびデータ連携仕様書を生成する。

2. **比較環境：従来型エージェント（対照群）**
    * ローカルGitリポジトリ（DDLファイル等）と、汎用的なファイル検索・Grep用MCPツールのみが与えられた標準的なコーディングエージェント環境。

## 6. 期待される成果
* **データカタログ主導アプローチの優位性証明:**
    「ローカルファイルの検索＋汎用ツールの組み合わせ」と比較して、セマンティックレイヤーを活用したアプローチが、エンタープライズにおけるコード生成の正確性・開発効率において圧倒的に優れていることを実証する。

* **推測（ハルシネーション）の排除:**
    業務要件と実装の乖離という、AI開発における最大の課題を「データカタログによる正確な意味の提供」というアーキテクチャによって解決できることを証明する。

* **高品質なドキュメントの自動生成:**
    コードだけでなく、誰もが理解できる「意味づけされたデータ仕様書」をコード開発と同時に自動生成する仕組みのベースラインを獲得する。
