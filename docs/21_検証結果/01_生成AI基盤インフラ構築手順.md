## 共通ネットワーク構築
```bash
cd "$AI_PLATFORM_POC_ROOT/infra/00-network"
docker compose up -d
```

この `00-network` では、`ai_platform_net` の作成に加えて、ローカル/同一Dockerネットワーク向けの HTTP(S) Proxy として Squid (`localhost:3128`, コンテナ内からは `squid:3128`) を提供します。

この `00-network` は、`ai_platform_internal`（`internal: true`）と `ai_platform_egress` を作成し、Squid を両ネットワークに接続します。

### (任意) Squid Proxy の疎通確認

ホストから確認する場合（`infra/00-network/docker-compose.yml` の `ports` を有効化している場合）:

```bash
curl -I -x http://localhost:3128 https://opencode.ai
```

コンテナ（`ai_platform_internal`）から確認する場合:

```bash
docker run --rm --network ai_platform_internal curlimages/curl:8.12.1 \
  -I -x http://squid:3128 https://opencode.ai
```

### (任意) Proxy を使う場合の環境変数

ホスト上で `uv` / `pip` / `curl` 等を実行する際に Proxy を通したい場合は、以下を設定します。

```bash
export HTTP_PROXY=http://localhost:3128
export HTTPS_PROXY=http://localhost:3128
export NO_PROXY=localhost,127.0.0.1,::1
```

Docker Compose の各サービスから外部通信を Proxy 経由にしたい場合は、対象サービスの `environment` に以下のように追加してください（コンテナ内からは `squid:3128` を参照します）。

```yaml
environment:
  - HTTP_PROXY=http://squid:3128
  - HTTPS_PROXY=http://squid:3128
  - NO_PROXY=localhost,127.0.0.1,::1
```

NOTE: Squid の挙動（許可する宛先/ポート等）は [infra/00-network/squid.conf](../../infra/00-network/squid.conf) で調整できます。

### 設定ファイルの作成

**`.env`**

```env
LITELLM_MASTER_KEY=sk-poc-master-key-12345
OPENAI_API_KEY=sk-your-openai-api-key
LANGFUSE_PUBLIC_KEY=pk-lf-... # Langfuseのキー
LANGFUSE_SECRET_KEY=sk-lf-... # Langfuseのキー

```

**`docker-compose.yml`**
このリポジトリの実体は [infra/05-litellm/docker-compose.yml](../../infra/05-litellm/docker-compose.yml) です。

- `LANGFUSE_HOST` は compose 側で `http://langfuse-web:3000` に固定しています（`.env` に定義は不要）

```yaml
services:
  litellm:
    restart: always
    image: ghcr.io/berriai/litellm:main-latest
    environment:
      - LITELLM_MASTER_KEY=${LITELLM_MASTER_KEY}
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - LANGFUSE_PUBLIC_KEY=${LANGFUSE_PUBLIC_KEY}
      - LANGFUSE_SECRET_KEY=${LANGFUSE_SECRET_KEY}
      - LANGFUSE_HOST=http://langfuse-web:3000
    volumes:
      - ./config.yaml:/app/config.yaml
      - ./src/ai_platform_litellm:/app/ai_platform_litellm
    ports:
      - "4000:4000"
    command: [ "--config", "/app/config.yaml", "--detailed_debug" ]
    extra_hosts:
      - "localhost:host-gateway"

networks:
  default:
    name: ai_platform_internal
    external: true

```
