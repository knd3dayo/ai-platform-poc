## 0. 前提

本書では、生成AIをエンタープライズの業務に安全かつ効果的に組み込むため、システム全体を以下の「3つの層」に分割して設計するアーキテクチャを採用しています。
ここで登場する各層の具体像は以下の前提資料で定義されていますが、本書を読み進める上での基本的な役割は以下の通りです。

### コア層
* **Application（アプリケーション）層：AIの「脳」**
* プロンプトやエージェント（WF型/自律型/SV型）を用いて、実際の業務ロジック、推論、タスクの進行管理（オーケストレーション）を担う層です。
* 詳細：[02_AIエージェントの業務適用を見据えた生成AIアプリケーション層の検討.md](./02_AIエージェントの業務適用を見据えた生成AIアプリケーション層の検討.md)


* **AIガバナンス層：AIの「免疫・安全網」**
* LLMの通信やツール実行の間に介在し、個人情報（PII）のマスキング、ハルシネーション（虚偽出力）の検知、監査ログの記録など、全社的なガバナンスと統制を強制する層です。
* 詳細：[04_AIエージェントの業務適用を見据えた生成AIガバナンス層の検討.md](./04_AIエージェントの業務適用を見据えた生成AIガバナンス層の検討.md)
* 本書では、この層を一貫して「**AIガバナンス層**」と呼称します。


* **Tool（ツール）層：AIの「手足・感覚器官」**
* 社内DBやSaaS等とやり取りするためのステートレスなインターフェース（MCP等）を提供する層です。統合認証基盤（IdP）や各種ゲートウェイ（データ仮想化、ゼロトラストプロキシ等）と連携し、実行ユーザーの権限を引き継ぐとともに、AIに「データの意味（セマンティック）」を正しく提供する役割を持ちます。
* 詳細：[03_AIエージェントの業務適用を見据えた生成AIツール層の検討.md](./03_AIエージェントの業務適用を見据えた生成AIツール層の検討.md)

---

### 周辺実行基盤（責務を成立させるための配置要素）

上記の「3層（Application / AIガバナンス / Tool）」は責務による分類です。
実際の業務適用では、これらの責務をシステムとして成立させるために、さらに以下の実行基盤や配置要素が必要になります。

* **BFF / 入口オーケストレーション**：Application層が要求する `trace_id` の確定、同期/非同期境界の制御、状態管理DBのI/O、非同期通知を north 境界で受け持つ実装コンポーネント
  * 詳細：主に [../02_アーキテクチャ実現方式/00_生成AI基盤のコンポーネント配置と実装・運用原則.md](../02_アーキテクチャ実現方式/00_生成AI基盤のコンポーネント配置と実装・運用原則.md) と [../02_アーキテクチャ実現方式/02_AIガバナンス層の実現方式.md](../02_アーキテクチャ実現方式/02_AIガバナンス層の実現方式.md) を参照

* **非同期HITL / 非同期連携基盤**：Application層の長時間処理、非同期HITL、Pause / Resume、外部Webhook駆動を疎結合で実現する実行基盤（DLQ/リトライ含む）
  * 詳細：主に [../02_アーキテクチャ実現方式/04_アプリケーション層の実現方式.md](../02_アーキテクチャ実現方式/04_アプリケーション層の実現方式.md) を参照
  * 旧参照入口：[AIエージェントの業務適用を見据えた非同期連携基盤（Event Bus）の検討.md](./AIエージェントの業務適用を見据えた非同期連携基盤（Event Bus）の検討.md)

* **Observability / 評価運用**：AIガバナンス層の横断能力として、`trace_id` をキーに止められる・追える・説明できる運用を支える実装要素
  * 詳細：主に [../02_アーキテクチャ実現方式/02_AIガバナンス層の実現方式.md](../02_アーキテクチャ実現方式/02_AIガバナンス層の実現方式.md) を参照
  * 旧文書の参照入口：[AIエージェントの業務適用を見据えた運用監視基盤（Observability）の検討.md](./AI%E3%82%A8%E3%83%BC%E3%82%B8%E3%82%A7%E3%83%B3%E3%83%88%E3%81%AE%E6%A5%AD%E5%8B%99%E9%81%A9%E7%94%A8%E3%82%92%E8%A6%8B%E6%8D%AE%E3%81%88%E3%81%9F%E9%81%8B%E7%94%A8%E7%9B%A3%E8%A6%96%E5%9F%BA%E7%9B%A4%EF%BC%88Observability%EF%BC%89%E3%81%AE%E6%A4%9C%E8%A8%8E.md)

* **API Gateway（任意の入口防御）**：認証・WAF・レート制限・経路制御などの決定論的な入口防御を共通インフラとして担う選択肢
  * 詳細：[../02_アーキテクチャ実現方式/04_アプリケーション層の実現方式.md](../02_アーキテクチャ実現方式/04_アプリケーション層の実現方式.md)
