# はじめに

本章の「アーキテクチャ検討」は、生成AIを業務へ適用する際に、何をどの責務に分けて設計すべきかを整理するための資料群です。狙いは、単にAIアプリを作ることではなく、これまで人手に依存してきた業務を、説明可能で統制可能なシステムとして成立させるための全体像を定義することにあります。

そのため本章では、まず [01_AIエージェントの業務適用を見据えた生成AIアーキテクチャ検討.md](./01_AIエージェントの業務適用を見据えた生成AIアーキテクチャ検討.md) で全体像を示し、その上で [02_AIエージェントの業務適用を見据えた生成AIアプリケーション層の検討.md](./02_AIエージェントの業務適用を見据えた生成AIアプリケーション層の検討.md)、[03_AIエージェントの業務適用を見据えた生成AIツール層の検討.md](./03_AIエージェントの業務適用を見据えた生成AIツール層の検討.md)、[04_AIエージェントの業務適用を見据えた生成AIガバナンス層の検討.md](./04_AIエージェントの業務適用を見据えた生成AIガバナンス層の検討.md) で各レイヤーを掘り下げます。

## 1. アーキテクチャ検討の概要

本資料群が扱う中心テーマは、次の問いです。

* なぜ企業には、今なおシステム化されず人手で残っている業務が多いのか。
* 生成AIやAIエージェントを使えば、その業務はどこまでシステム化できるのか。
* そのとき、業務ロジック、外部システム接続、統制・監査をどのように分離して設計すべきか。

本章では、この問いに対する答えとして、生成AI基盤を大きく **Application層 / Tool層 / AIガバナンス層** の3層に分けて捉えます。これは単なる概念整理ではなく、AIの思考、外界への作用、企業としての統制を混在させず、それぞれを独立した責務として扱うための設計原則です。

## 2. 01〜04の読み方と役割分担

### 2.1 01: 全体像を定義する文書

[01_AIエージェントの業務適用を見据えた生成AIアーキテクチャ検討.md](./01_AIエージェントの業務適用を見据えた生成AIアーキテクチャ検討.md) は、本章全体の総論です。主に以下を記載しています。

* その背景にある「技術の壁」「投資の壁」「統制の壁」「暗黙知の壁」とは何か。

### 2.4 04: AIガバナンス層を扱う文書

[04_AIエージェントの業務適用を見据えた生成AIガバナンス層の検討.md](./04_AIエージェントの業務適用を見据えた生成AIガバナンス層の検討.md) は、生成AI活用に伴う統制、評価、監査を担う AIガバナンス層を扱います。主に以下を記載しています。

## 3. 本章で扱う3層の全体像

01〜04を通して扱うコアのアーキテクチャは、次の3層です。

* **Application（アプリケーション）層**: AIの推論、計画、業務フロー制御を担う「脳」。
* **Tool（ツール）層**: DB、SaaS、ファイル、社内システムとの接続を担う「手足」。
* **AIガバナンス層**: 入出力統制、根拠性、監査、評価、停止判断を担う「免疫・安全網」。

本章では、この3層を分けることで、業務ロジックの変更、接続先システムの変更、統制要件の変更が互いに波及しにくい構造を目指します。以降の文書は、この3層それぞれの成立条件と責務境界を定義するためのものです。

## 4. 3層を成立させる周辺実行基盤

3層は責務の分類ですが、実際の業務システムとして成立させるには周辺実行基盤も必要です。本章の後続資料や実現方式資料では、特に以下の配置要素を前提にします。

* **BFF / 入口オーケストレーション**: `trace_id` の確定、同期/非同期境界の制御、状態管理DBのI/O、通知制御を受け持つ入口コンポーネント。
* **非同期HITL / 非同期連携基盤**: 長時間実行、Pause / Resume、人間承認、Webhook連携を支える基盤。
* **Observability / 評価運用**: `trace_id` を軸に、止められる・追える・説明できる運用を実現する基盤。
* **API Gateway**: 認証、WAF、レート制限などの決定論的な入口防御を担う共通基盤。

これらの詳細は、主に [../02_アーキテクチャ実現方式/01_生成AI基盤のコンポーネント配置と実装・運用原則.md](../02_アーキテクチャ実現方式/01_生成AI基盤のコンポーネント配置と実装・運用原則.md)、[../02_アーキテクチャ実現方式/02_生成AIアプリケーション層の実現方式.md](../02_アーキテクチャ実現方式/02_生成AIアプリケーション層の実現方式.md)、[../02_アーキテクチャ実現方式/03_生成AIツール層の実現方式.md](../02_アーキテクチャ実現方式/03_生成AIツール層の実現方式.md)、[../02_アーキテクチャ実現方式/04_生成AIガバナンス層の実現方式.md](../02_アーキテクチャ実現方式/04_生成AIガバナンス層の実現方式.md) を参照してください。
