# 生成AI基盤インフラ構築手順

* Docker環境のセットアップ: 後述のソフトウェア、アプリケーションの実行環境の構築

* PostgreSQLのセットアップ: Langfuse、Dify、litellm、langgraph、状態管理DBなどで使用するDBとして用意。
* Redisのプロビジョニング: Event Bus、Langfuse、Difyなどで使用するEvent Buとして用意。

* Langfuseのセットアップ: 全ての実験ログがここに入るようにする。
* Difyのセットアップ: 「WF型」のアプリケーション実行環境。
* LiteLLM Proxyのデプロイ: 外部LLM（Azure OpenAI等）への接続を集約し、APIキーを隠蔽。

* サンドボックス環境のセットアップ: 「自律型」のアプリケーション実行環境。
* (PoCのMCPバックエンド用): Denodo環境構築
* (PoC前のテスト用): IdP(Zitadel)環境構築

### 2-2. 設定ファイルの作成

**`2_litellm/.env`**

```env
LITELLM_MASTER_KEY=sk-poc-master-key-12345
OPENAI_API_KEY=sk-your-openai-api-key
LANGFUSE_PUBLIC_KEY=pk-lf-... # Langfuseのキー
LANGFUSE_SECRET_KEY=sk-lf-... # Langfuseのキー

```

**`docker-compose.yml`**
公式Langfuse構成に合わせ、`LANGFUSE_HOST` はサービス名である **`langfuse-web`** を指定します。

```yaml
services:
  litellm:
    image: ghcr.io/berriai/litellm:main-latest
    environment:
      - LITELLM_MASTER_KEY=${LITELLM_MASTER_KEY}
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - LANGFUSE_PUBLIC_KEY=${LANGFUSE_PUBLIC_KEY}
      - LANGFUSE_SECRET_KEY=${LANGFUSE_SECRET_KEY}
      - LANGFUSE_HOST=${LANGFUSE_HOST} 
    volumes:
      - ./config.yaml:/app/config.yaml
      - ./src/ai_platform_litellm:/app/ai_platform_litellm
    ports:
      - "4000:4000"
    command: [ "--config", "/app/config.yaml" ]
    networks:
      - ai_platform_net

networks:
  ai_platform_net:
    external: true

```
### 起動
```bash
docker compose up -d
```

# ZitadelでのPythonテストクライアント用のキー作成手順

まずは、Zitadelのコンソールで **Projects** ➔ **`AI-Platform`** （作成済みのプロジェクト）を開いてください。

### 設定1：Pythonスクリプト用（Machine-to-Machine）

ブラウザを介さず、スクリプトが直接認証するための設定です。

1. **Applications** ➔ **Create** をクリック。
2. **Name**: `mcp-test-client`
3. **Type**: **`API`** を選択。
4. **Auth Method**: **`Basic`** を選択。
5. **情報の保存**:
* 表示される **`Client ID`** と **`Client Secret`** をメモし、Pythonコードの変数にセットします。

## ステップ4：DifyとLiteLLM Proxyの接続

1. `http://localhost:8080` (Dify) にアクセス。
2. **Settings > Model Providers > OpenAI-API-compatible** を追加。
* **Endpoint URL**: `http://litellm:4000/v1`
* **API Key**: `sk-poc-master-key-12345`

Zitadelを選択いただきありがとうございます。ZitadelはGo製でリソース消費が少なく、かつ設定画面も直感的なため、今回のPoC環境には最適です。

Zitadelを認証基盤（IdP）として、LiteLLMをMCP Gateway（リソースサーバー）として構築し、PythonクライアントからOIDC（OAuth 2.1）で認証・アクセスを行う一連の手順をまとめました。
