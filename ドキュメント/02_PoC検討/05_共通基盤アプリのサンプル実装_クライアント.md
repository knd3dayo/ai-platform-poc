実業務（サポート業務）での利用を想定し、これまで構築してきた「SSEによる実況中継」と「認証伝搬」を最大限に活かすなら、フレームワークは **React (Vite + Tailwind CSS)** が最もおすすめです。

### なぜ React なのか？

* **Streamlit**: Pythonだけで書けて非常に「簡単」ですが、ページ全体が再描画（Rerun）される特性上、**複雑なSSE（ストリーミング）の制御や、チャット画面の細かいスクロール制御が苦手**です。
* **React**: 状態管理（State）が強力で、AIが考えている途中のメッセージを「逐次追記」したり、特定の `trace_id` だけをリアルタイムにリッスンする動きが非常にスムーズに実装できます。

### 2. チャット画面の実装案

BFF（localhost:8000）に接続し、SSEを受け取って画面を更新する中心的なコンポーネントです。

**`src/App.jsx`**

```jsx
import React, { useState, useEffect, useRef } from 'react';
import { Send, Bot, User, Loader2 } from 'lucide-react';

const App = () => {
  const [messages, setMessages] = useState([]);
  const [input, setInput] = useState('');
  const [isThinking, setIsThinking] = useState(false);
  const [traceId, setTraceId] = useState("tr-initial-123"); // 実際はBFFから取得
  const scrollRef = useRef(null);

  // 1. SSE接続の設定
  useEffect(() => {
    if (!isThinking) return;

    // BFFのSSEエンドポイントに接続
    const eventSource = new EventSource(`http://localhost:8000/ai/events/${traceId}`);

    eventSource.onmessage = (event) => {
      const data = JSON.parse(event.data);
      
      if (data.type === 'HEARTBEAT') return;

      // AIの思考や回答をメッセージリストに追加・更新
      setMessages((prev) => {
        const lastMsg = prev[prev.length - 1];
        if (lastMsg?.type === 'AGENT_THOUGHT' && data.event_type === 'AGENT_THOUGHT') {
          // 同じ思考プロセスなら上書きまたは追記して「動いている感」を出す
          return [...prev.slice(0, -1), { ...data, sender: 'bot' }];
        }
        return [...prev, { ...data, sender: 'bot' }];
      });

      if (data.event_type === 'FINAL_ANSWER') {
        setIsThinking(false);
        eventSource.close();
      }
    };

    eventSource.onerror = () => {
      eventSource.close();
      setIsThinking(false);
    };

    return () => eventSource.close();
  }, [isThinking, traceId]);

  // メッセージ追加時に自動スクロール
  useEffect(() => {
    scrollRef.current?.scrollIntoView({ behavior: 'smooth' });
  }, [messages]);

  const sendMessage = async () => {
    if (!input.trim()) return;
    const userMsg = { sender: 'user', message: input };
    setMessages([...messages, userMsg]);
    setInput('');
    setIsThinking(true);

    // 2. 本来はここでBFFにPOSTリクエストを送り、LangGraphを起動させる
    // await fetch('http://localhost:8000/ai/chat', { ... });
  };

  return (
    <div className="flex flex-col h-screen bg-gray-50">
      {/* ヘッダー */}
      <header className="bg-white border-b p-4 flex justify-between items-center">
        <h1 className="text-xl font-bold text-blue-600 flex items-center gap-2">
          <Bot /> Support AI Assistant
        </h1>
        <span className="text-xs text-gray-400">Trace: {traceId}</span>
      </header>

      {/* チャットエリア */}
      <main className="flex-1 overflow-y-auto p-4 space-y-4">
        {messages.map((msg, idx) => (
          <div key={idx} className={`flex ${msg.sender === 'user' ? 'justify-end' : 'justify-start'}`}>
            <div className={`max-w-[80%] p-3 rounded-lg ${
              msg.sender === 'user' ? 'bg-blue-600 text-white' : 'bg-white border text-gray-800'
            }`}>
              <p className="text-sm">{msg.message || msg.payload?.message}</p>
              {msg.event_type === 'AGENT_THOUGHT' && (
                <span className="text-[10px] text-orange-400 mt-1 block">● AI is thinking...</span>
              )}
            </div>
          </div>
        ))}
        <div ref={scrollRef} />
      </main>

      {/* 入力エリア */}
      <footer className="p-4 bg-white border-t">
        <div className="flex gap-2">
          <input
            className="flex-1 border rounded-full px-4 py-2 focus:outline-none focus:ring-2 focus:ring-blue-500"
            placeholder="サポート内容を入力..."
            value={input}
            onChange={(e) => setInput(e.target.value)}
            onKeyPress={(e) => e.key === 'Enter' && sendMessage()}
          />
          <button 
            onClick={sendMessage}
            disabled={isThinking}
            className="bg-blue-600 text-white p-2 rounded-full hover:bg-blue-700 disabled:bg-gray-400"
          >
            {isThinking ? <Loader2 className="animate-spin" /> : <Send size={20} />}
          </button>
        </div>
      </footer>
    </div>
  );
};

export default App;

```

---

## この実装の「実務的な」ポイント

1. **EventSource による自動更新**:
`useEffect` 内で `EventSource` を作成することで、BFFから Redis Streams のデータが届くたびに、画面が勝手に更新されます。ユーザーは「更新ボタン」を押す必要がありません。
2. **思考プロセスの可視化**:
`AGENT_THOUGHT` というイベントタイプを見て、オレンジ色のラベルを表示しています。これにより、AIが裏で Denodo を叩いている間も、ユーザーは「フリーズしていない」と安心できます。
3. **状態の分離**:
「過去の回答」と「現在進行中の思考」を配列内で区別して管理できるため、複雑な業務フローでも表示が崩れません。
